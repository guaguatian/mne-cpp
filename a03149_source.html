6 Getting Qt
7 ----------
8 
9 * Download the current Qt version:
10  Go to the Qt download section and download the [Qt installer](https://www.qt.io/download-qt-installer?hsCtaTracking=9f6a2170-a938-42df-a8e2-a9f0b1d6cdce%7C6cb0de4f-9bb5-4778-ab02-bfb62735f3e5). Qt 5.10.0 or higher is needed in order to have full Qt3D support.
11 
12  Windows users: Make sure NOT to download the WinRt or UWP version.
13 
14  Select the appropriate version based on your compiler and development platform.
15 
16  For example if you are using the Microsoft Visual Studio 2015 Compiler on a Windows 32bit system, select the "MSVC 2015 32-bit" version.
17 
18 * Install the Qt version with the minimum of the following features (uncheck all other boxes):
19 
20  - Qt/5.10.0/Pre-compiled Qt (i.e. Qt 5.10.0/MSVC 2015 32-bit)
21  - Qt/5.10.0/QtCharts
22  - Qt/Tools/QtCreator
23 
24 * You now have the latest Qt version installed on your local machine. Add the Qt bin folder to the environment variable:
25 
26  - Windows:
27 
28  - Right-click Computer and click Properties.
29  - Click Advanced System Settings link in the left column.
30  - In the Advanced window and click the Environment Variables button.
31  - Add to the `PATH` variable `C:\Qt\5.10.0\msvc2015\bin` (the qt bin path may differ depending on your Qt installation).
32 
33  - Linux:
34 
35  - Add Qt bin and lib directory to the environment variables (the qt paths may differ depending on your Qt installation).
36  by adding it to your `.bashrc` file under `/home/<username>/`:
37 
38  $ export PATH=$PATH:~/Qt/Qt5.10.0/bin
39  $ export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:~/Qt/Qt5.10.0/lib
40 
41  Note this command may differ depending on your shell (for example `$ setenv` in `.tchrc`)
42 
43 * In case you work on a Windows machine and do not have a compiler set up:
44 
45  - Go to the [Windows SDK download website](https://dev.windows.com/de-de/downloads/windows-8-1-sdk)
46  - Download and install the Windows SDK (during the installation, make sure to also select the Debugging Tools for Windows box).
47  - Open the QtCreator and select the debugger under `Tools/Options/Build & Run/Kits/[your Kit name goes here]/Debugger`.

49 Get the MNE-CPP source code
50 ---------------------------
51 
52 Fork MNE-CPP's main repository (mne-tools/mne-cpp) to your own GitHub account:
53 
54 * Go to the GitHub website and sign into your GitHub account.
55 * Go to the MNE-CPP's GitHub website.
56 * Click on the upper right Fork button. This will copy the main repository to your own GitHub account. This is the code you can now work with, without breaking the project's main code.
57 
58 Clone the forked MNE-CPP repository to your local machine:
59 
60 * Open the Git command line tool (Git CMD).
61 * Navigate to your folder where you want to clone the repository to (Use cd to navigate to a specific path on your hard drive). Please note that the folder path should not contain any whitespaces. For example, `C:/Git/` would be a good candidate to clone the repository to.
62 * Type in the following command to clone the repository:
63 
64  $ git clone https://github.com/<YourGitUserName>/mne-cpp.git
65 
66 * Every time you want to pull the newest changes made in the main MNE-CPP repository use the code below. Please make sure that you are on the local `master` branch when you do this:
67 
68  $ git pull https://github.com/mne-tools/mne-cpp.git master
69 
70 Congrats, you now have the latest MNE-CPP source code on your remote and local machine. For a more detailed overview of how to make a pull request, we recommend checking out the [guide on the official Git website](https://git-scm.com/book/en/v2/GitHub-Contributing-to-a-Project).
71 
72 Compile the MNE-CPP source code
73 -------------------------------
74 
75 * If you are working on an operating system on a "non-western" system, i.e. Japan, you might encounter problems with unicode interpretation. Please do the following: Go to Control Panel -> Language and Region -> Management tab -> Language Settings for non-Unicode Programs -> Set to English (U.S.) -> Reboot your system.
76 * Go to your cloned repository folder and run the `mne-cpp/mne-cpp.pro` file with the QtCreator.
77 * The first time you open the mne-cpp.pro file with QtCreator you will be prompted to configure the project with an installed Qt version. Select the appropriate Qt version, e.g. Qt MSVC 2015 32-bit and configure the project.
78 * In QtCreator select the Release mode in the lower left corner.
79 * In the Qt Creator's Projects window, right mouse click on the top level mne-cpp tree item and select Run qmake. Wait until progress bar in lower right corner turns green (this step may take some time).
80 * Right mouse click again and then hit Build (this step may take some time). Wait until progress bar in lower right corner turns green.
81 * After the build process is finished, go to the `mne-cpp/bin` folder. All applications and libraries should have been created throughout the build process.
82 * In order to run the examples you must download the MNE-Sample-Data-Set from [here](https://osf.io/86qa2/download ) and extract the files to `mne-cpp/bin/MNE-sample-data`
